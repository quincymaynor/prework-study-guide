HTML

  • The head element contains information about the webpage
  • Common elements in the head: meta-related, content division, and semantic
  • The body element represents the visible content shown to the user
  • Common elements in the body: text, image, header, and list
  • style defines the CSS styles associated with a website
  • base defines the base URL for a webpage
  • link connects an external resource to the HTML document
  • rel specifies the relationship between the current and linked document
  • href specifies the location (URL) of the external resource
  • meta defines metadata such as the character set, description, keywords, author, and viewport
  • script embeds client-side scripts in the HTML document
  • h1 indicates the largest header on the page and h6 indicates the smallest
  • p indicates a paragraph or block of text
  • ul, ol, and li represent unordered lists, ordered lists, and list items
  • img contains information about images displayed on the webpage
  • br creates a line break between two blocks of content
  • a stands for anchor and creates links to the same or other pages
  • Types of information that can be stored in the meta tag
  • UTF-8 and character encoding

CSS

  • CSS Rule: selector {declaration block}
  • {declaration block} contains one or more declarations
  • Declarations contain a property and value, structured as property: value;
  • The * selector is used to indicate that the following declaration block applies to every element on the page
  • A . is used to designate a class selector, ex. .card means that any HTML element with class="card" will be affected by the properties and values in the declaration block
  • A margin indicates the space around the OUTSIDE of an element
  • A padding indicates the space around the content INSIDE of an element
  • The CSS box model
  • CSS tutorials and references

Git

  • git status: checks what branch we are currently on
  • git checkout main: switches user to the main branch
  • git checkout -b [branch name]: creates new branch and switches to it
  • git add -A: adds all changes in the working branch to the staging area
  • git commit -m "commit description": commits the previously staged changes with a message attached
  • git pull origin main: pulls down any changes from the base branch
  • git push origin [branch name]: pushes local changes to the remote branch
  • Default branch is called the main branch
  • Feature branch is the copy of the main branch that all changes are commited to for approval before merging
  • A pull request (PR) is made to merge the featured branch with the main
  • Check for an existing SSH key, Generate a new SSH key, Add SSH key to the ssh-agent, Add SSH key to GitHub account
